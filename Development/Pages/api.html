GET Request (HTTP)

Send SMS using HTTP GET method:

http://1.1.1.1:8001/api?
  username=<username>&
  password=<password>&
  ani=<ani>&
  dnis=<dnis>&
  message=<message>&
  command=submit&
  serviceType=<serviceType>&
  longMessageMode=<longMessageMode>

GET Request (HTTPS)

Use HTTPS for secure communication:

https://domainname:8002/api?
  username=<username>&
  password=<password>&
  ani=<ani>&
  dnis=<dnis>&
  message=<message>&
  command=submit&
  serviceType=<serviceType>&
  longMessageMode=<longMessageMode>

POST Request (credentials in URL)

Send JSON payload with credentials in the URL: 

curl -H "Content-Type: application/json" -X POST \
  -d '{"ani":"ani","dnis":"dnis","message":"test"}' \
  "http://1.1.1.1:8001/api?command=submit&username=username&password=password"

POST Request (credentials in JSON body)

Send everything in JSON format including credentials: 

curl -H "Content-Type: application/json" -X POST \
  -d '{
    "username": "username",
    "password": "password",
    "command": "submit",
    "ani": "ani",
    "dnis": "dnis",
    "message": "test"
  }' \
  "http://1.1.1.1:8001/api"
Note: To send multiple SMS messages, add several comma-separated DNIS to the <dnis> field.
Parameter Description Required
command Request type. Possible values: submit, query, mo. To send a message over HTTP, specify command=submit. Yes
dnis Destination number. Must be in international E.164 format. Max 15 digits. Length limit: 2048 symbols or 100 numbers. If exceeded, returns 400 Bad Request. Cannot use multiple for longMessageMode=split_or_payload. Yes
message Message text. Yes
password Account password. Yes
serviceType Service type for registered interconnection. Can be blank. Max length: 9 bytes. Yes
username Account login. Yes
ani Caller ID. Alphanumeric up to 32 characters. May vary by route. No
dataCoding Encoding scheme (int). Options: 0 (default), 1 (ASCII), 3 (Latin1), 6 (Cyrillic), 7 (Hebrew), 8 (UCS2), 10 (ISO-2022-JP), 14 (KSC5601). No
esmClass SMPP esm_class value. Integer: 0–255. No
flash 0 (regular) or 1 (flash). Flash messages display on screen and aren't stored. No
longMessageMode Type of long message processing:
  • cut: Trim to 140 bytes.
  • split or split_sar: Use UDH or SAR headers.
  • single_id_split: Common message ID for all parts.
  • payload: Sends text via message_payload.
  • split_or_payload: Hybrid, not for multiple DNIS.
Splitting depends on dataCoding. Default: cut. 		No
incMsgId Client message ID (max 64 chars). Used to correlate split messages with suffixes like -1, -2. No
srcTon, srcNpi, dstTon, dstNpi SMPP address parameters (Sender ID and Destination number). Integers. No
priorityFlag SMPP priority flag. 0 or 1. No
registeredDelivery SMPP delivery receipt flag. 0 or 1. No
replaceIfPresentFlag SMPP replace flag. 0 or 1. No
silent Send silent SMS. 0 = not silent. Any non-zero = silent.
Message is not shown or notified on user device (depends on vendor). 		Yes

✅ Response Format

When the request is successfully processed, the server responds with:

HTTP/1.1 200 OK Content-Type: application/json { "message_id": "alss-a1b2c3d4-e5f67890" }

📦 Response with Multiple Segments

This response occurs when:

  • The request contains multiple DNIS values (comma-separated)
  • longMessageMode = split or split_sar and the message is too long
HTTP/1.1 200 OK Content-Type: application/json [ { "dnis": "34511121", "message_id": "5b4c46a8-8dc9-44b4-f55f-3bef56819305", "segment_num": "1" }, { "dnis": "34511121", "message_id": "5b4c46a8-46bc-7ee6-4a16-7d4e5a0d14af", "segment_num": "2" } ]

❌ Rejection Scenarios

1. No Compatible Routes Found

HTTP/1.1 400 Bad Request Content-Type: text/html; charset=UTF-8 NO ROUTES

2. Invalid Username or Password

HTTP/1.1 401 Unauthorized Content-Type: text/html; charset=UTF-8 not authorized (check login and password)

3. Service Type Too Long

HTTP/1.1 400 Bad Request Content-Type: text/html; charset=UTF-8 Service type is invalid

4. Message Detected as a Loop

HTTP/1.1 508 Loop Detected Content-Type: text/html; charset=UTF-8 loop detected

📩 SMS Status Request

The links below are intended for requesting delivery reports over HTTP from the client side.

🔗 Request Format:

http://1.1.1.1:8001/api?username=<username>&password=<password>&messageId=<messageId>&command=query

📌 Parameters

Parameter Description
username Login (user identification)
password Password for authentication
messageId Message identifier received with the submission response
command Request type. Must be set to query


All parameters are obligatory.


Response format:


In case of successful processing, the status in the header of the HTTP response is 200 OK. The response body contains the one of the following possible values:

Status Code Description
ENROUTE Message is in routing stage. This status may be returned while the message is in SENT status.
DELIVRD Message was successfully delivered to the subscriber.
EXPIRED Message storage period has expired without successful delivery.
DELETED Message was deleted before delivery could occur.
UNDELIV Message could not be delivered.
ACCEPTD Message has been accepted by the SMSC for delivery.
UNKNOWN Message status is unknown. This may happen if the client queries too late, after data has been cleared from the memory (default TTL is 24h).
REJECTD Message was rejected by the SMSC. Can also indicate internal failures (e.g., NO ROUTES).
IM_EXPD Delivery report not received from the IM provider within the configured im_ttl timeout.

📩 SMS Status Response

Below are the possible responses returned when querying SMS delivery status:

✅ Successful Response Example

HTTP/1.1 200 OK
Content-Type: application/json

{
  "status": "DELIVRD",
  "delivery_time": "20210922093309",
  "mccmnc": "214099",
  "error_code": "000",
  "system_delivery_time": "210922093309"
}

Explanation of fields:

  • delivery_time: Time when message was delivered (format: YYYYMMDDHHMMSS)
  • mccmnc: E212 code identifying the mobile network
  • error_code: Delivery error code
  • system_delivery_time: Time when delivery report was received by the system (YYMMDDHHMMSS)

📄 Unknown Status Example

HTTP/1.1 200 OK
Content-Type: application/json

{
  "status": "UNKNOWN",
  "delivery_time": "",
  "mccmnc": ""
}

❌ Unauthorized Access

HTTP/1.1 401 Unauthorized
Content-Type: text/html; charset=UTF-8

not authorized (check login and password)

⚠️ Invalid service_type Length

HTTP/1.1 400 Bad Request
Content-Type: text/html; charset=UTF-8

Service type is invalid
